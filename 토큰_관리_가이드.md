## 🔒 안전한 토큰 저장 방법

### 방법 1: Windows 자격 증명 관리자 (권장)

Git이 자동으로 Windows 자격 증명 관리자에 토큰을 저장합니다.

**확인 방법:**
1. 제어판 → 자격 증명 관리자
2. Windows 자격 증명
3. `git:https://github.com` 항목 확인

**장점:**
- ✅ 자동으로 저장됨
- ✅ 암호화되어 저장됨
- ✅ Git 명령어 사용 시 자동 인증

### 방법 2: .env 파일에 저장 (선택사항)

**주의:** `.env` 파일은 이미 `.gitignore`에 포함되어 있어 Git에 업로드되지 않습니다.

#### A. .env 파일 생성
```bash
# .env.example 파일을 참고하여 .env 파일 생성
# .env 파일에 다음 내용 추가 (실제 토큰 값 사용):
GITHUB_TOKEN=your_github_token_here
```

#### B. .env 파일 사용 (스크립트에서)
```javascript
// Node.js에서 사용 예시
require('dotenv').config();
const token = process.env.GITHUB_TOKEN;
```

## 🚀 추후 작업 방법

### 1. 변경사항 업로드 (가장 자주 사용)

```bash
cd C:\test\test02

# 변경사항 확인
git status

# 모든 변경사항 추가
git add .

# 커밋 생성
git commit -m "변경 내용 설명"

# GitHub에 푸시 (토큰 자동 사용)
git push origin main
```

**커밋 메시지 예시:**
```bash
git commit -m "feat: 새로운 기능 추가"
git commit -m "fix: 버그 수정"
git commit -m "docs: 문서 업데이트"
git commit -m "style: 코드 포맷팅"
```

### 2. 원격 저장소에서 가져오기

```bash
# 최신 변경사항 가져오기
git pull origin main

# 또는 fetch + merge
git fetch origin
git merge origin/main
```

### 3. 새 브랜치 작업

```bash
# 새 브랜치 생성 및 전환
git checkout -b feature/새기능

# 작업 후 커밋
git add .
git commit -m "feat: 새 기능 구현"
git push origin feature/새기능

# main 브랜치로 병합
git checkout main
git merge feature/새기능
git push origin main
```

## 🔄 토큰 갱신

토큰이 만료되면:

1. **새 토큰 생성**
   - 링크: https://github.com/settings/tokens
   - 기존 토큰 삭제 후 새 토큰 생성

2. **자격 증명 업데이트**
   - 제어판 → 자격 증명 관리자
   - `git:https://github.com` 항목 수정 또는 삭제
   - 다음 `git push` 시 새 토큰 입력
